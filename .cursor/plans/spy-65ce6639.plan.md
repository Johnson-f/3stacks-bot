# Plan: SPY options pinger

- Add options fetch helpers in [`src/service/finance/mod.rs`](src/service/finance/mod.rs) (or a new options helper module) that wrap `finance_query_core` to: list expirations, pick today’s expiration only, fetch that chain, compute spot, and return the top 5 calls above spot and top 5 puts below spot.
- Add a formatter in a new automation module (e.g., [`src/service/automation/options_data/mod.rs`](src/service/automation/options_data/mod.rs)) to shape the message text (strike, LTP, bid/ask, IV, OI/Vol, ITM) and orchestrate the fetch logic.
- Wire an interval task (15 min) at startup (likely in [`src/main.rs`](src/main.rs) or a dedicated automation runner) that reads `OPTIONS_CHANNEL_ID` from env, and posts the formatted calls/puts for SPY when today’s expiration exists; if not present, log/skip with a brief notice to the channel.
- Keep errors user-friendly (e.g., “no expiration for today” or fetch/parse errors) and avoid crashing the bot.
- Optionally gate the automation behind an env flag (e.g., `ENABLE_OPTIONS_PINGER`) to turn it on/off without code changes.
